## Fix Phase

### Files Modified (all compile clean, 67/67 tests pass)

#### `tools/train/self_train.rail`
- **verify()**: captures exit code properly via separate files, catches segfaults/timeouts
- **harvest()**: SHA-256 full-content dedup (was 80-char fingerprint), uses `harvest_hashes.txt`
- **harvest()**: trivial filter — rejects programs with ≤3 non-blank/non-comment lines
- **Round counter**: increments `round + 1` before writing progress (was never incrementing)
- **Advancement threshold**: 5→3 consecutive 80%+ rounds (40% faster curriculum progression)
- **Fallback threshold**: 3→2 consecutive 0% rounds (wastes fewer bad LLM calls)
- **Task shuffling**: seeds shuffled via `shuf` before selection — breaks memorization
- **Server retry**: max 6 attempts with 30s backoff (was infinite recursion)

#### `tools/train/build_training_data.rail`
- **Variant reduction**: 15x → 3x (write_variants produces 3 entries, not 15)
- **Dynamic split**: computes 90/5/5 from actual line count (was hardcoded at 936/50/rest)
- **Regenerated**: 866 → 274 entries (592 pure-waste entries eliminated)

#### `tools/train/harvest_git.py`
- **Variant reduction**: 3x → 1x (make_variants returns single entry)
- **Regenerated**: 204 → 150 entries

#### `flywheel/train_cuda.py` (now v3)
- **Default seq_len**: 256 → 512 (covers P90 of data)
- **Validation eval**: every report_every steps on valid.jsonl
- **Overfit warning**: prints `!!OVERFIT` when val > 1.5x train
- **Quality filtering**: inline no-main check, trivial filter, oversized filter, SHA-256 dedup
- **UTF-8 encoding**: explicit on all file reads (was crashing on Windows)
- **Layer logging**: prints found vs targeted attention layers
- **Defaults tuned**: batch=1, lr=1.5e-5, iters=3000, warmup=150 (for 8GB VRAM)
- Deployed to Razer as `train_cuda_v3.py` + `run_v3.bat`

#### `flywheel/bench.rail`
- **Output verification**: verify() now checks stdout against expected output
- **Task format**: tab-separated `task\texpected` (bands 1-4 have exact expected values)
- **try_task**: passes expected to verify, feeds wrong_output back on retry
